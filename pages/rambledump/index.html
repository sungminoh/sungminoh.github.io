📲 설치

이 빌드에 들어간 것 (Finish Sync Architecture)

  • A즉시·내구성 읽기 — 글을 열 때 iCloud 다운로드를 기다리지 않고 캐시에서 즉시 표시. 한 번 본 글은 오프라인에서도 즉시 열림. 첫 탭 무한 스핀 제거(미스 시 30초 bounded).
  • B다른 기기 편집 반영 — 맥·다른 폰·Files·Obsidian에서 고친 글이 자동 반영. 열어둔 글이면 "외부에서 수정됨 [다시 불러오기]" 배너.
  • C충돌 시 데이터 보존(keep-both) — 두 기기가 같은 글을 따로 고치거나, 편집 중 외부 변경이 와도 둘 다 보존하고 충돌 해결 화면 표시. 예전처럼 한쪽이 조용히 사라지지 않음.
  • perf런치 흰 화면 단축 — iCloud 루트 경로 캐시 → 첫 설치 직후 1회(≤2초)만 흰 화면, 그 다음 실행부터 빠름.
  • perf글 열기 빠르게 — 충돌 체크 iCloud I/O를 메인 스레드에서 분리 → 모든 글 열기가 더 이상 멈칫하지 않음.

테스트 체크리스트

  • ① 런치: 첫 설치 직후 1회만 흰 화면 잠깐 → 앱 껐다 다시 켜면 흰 화면 거의 없이 빠르게 떠야 함.
  • ② 글 열기: 같은 글을 두 번째 열 때 즉시 / 비행기모드 켜고 이미 본 글 열어도 즉시.
  • ③ 원격 반영: 폰에서 글 A 열어둔 채, 맥/Files에서 같은 글 수정·저장 → 잠시 후 폰에 "외부에서 수정됨" 배너 → [다시 불러오기]로 새 내용.
  • ④ 오프라인 분기→재연결: 폰·맥 둘 다 비행기모드 → 같은 글을 서로 다르게 수정·저장 → 둘 다 온라인 복귀 → 그 글 열면 충돌 해결 화면(양쪽 버전, 둘 다 보존).
  • ⑤ 편집 중 외부 변경: 폰에서 글 편집 중 맥에서 같은 글 수정 → 폰에서 저장 → 내 편집 저장 + 맥 버전은 충돌본으로 보존.

③④⑤는 실제 iCloud 동기화가 필요한 시나리오입니다. 등록된 기기(성민 iPhone/iPad)에서만 설치됩니다.
