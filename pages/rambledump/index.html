이 빌드에 들어간 것

  • 🆕 빌드 8 — "글 계속 열면 멈춤" 근본 해결(완성): 글을 몇 번 열다 보면 앱 전체가 멈추고(글 스피너·풀다운·상단 버튼 윤곽만) 무한 대기하던 문제를 뿌리째 수정. 원인: 화면(리스트·글 열기·검색·새로고침)의 DB 읽기가 메인 스레드에서 실행됐는데, DB 읽기 풀 슬롯(5개)이 백그라운드 작업으로 다 차면 메인 스레드가 슬롯을 무한 대기 → 앱 전체 정지. 수정: 모든 화면 DB 읽기를 메인 밖(비동기)으로 옮기고, "메인에서 DB 읽기 금지" 가드를 코드에 영구 설치해 다시는 새 경로가 새지 않게 함. (빌드 7은 검색 경로를 빠뜨려 미완성 → 독립 리뷰가 적발 → 이번에 완성. 636개 테스트 + 가드로 검증 후 배포.)
  • 빌드 7 — DB 동시 읽기(DatabasePool/WAL): SQLite를 직렬 큐 → DatabasePool로 바꿔 읽기가 쓰기를 안 기다림. (빌드 8과 함께 작동.)
  • 빌드 4 — 재오픈 무한 스피너 수정: 글을 열어 편집했다가 뒤로 갔다 같은 글을 다시 누르면 무한 스피너에 걸리던 버그 수정(죽은 로드 상태가 재시도되지 않던 근본 원인 + 화면 재진입 시 로드 재발화). 코드 리뷰 지적(메인 스레드 잔여 I/O·검증 강화)도 함께 반영.
  • 반응성 제1원칙(INV-PERF) — UI/메인 스레드가 I/O로 멈추지 않도록 전면 수정:
    • 앱 첫 실행 흰 화면 제거 — 런타임(iCloud 해석·DB 마이그레이션·부트스트랩)을 백그라운드에서 비동기 초기화, 첫 프레임은 즉시 셸 렌더.
    • 글 열기 지연 제거 — 모든 글을 로컬 캐시로 backfill, 열기 경로 파일 읽기를 메인 스레드 밖으로(캐시 우선).
    • 타이핑 랙 제거 — 편집 중 자동저장이 리스트 전역 갱신을 유발하던 경로 차단(편집 종료 시 1회 갱신).
    • Settings/캡처 시트 열기·인덱스 조회의 메인 스레드 파일 I/O 제거, 포그라운드 전체 재색인 → 증분 동기로 전환.
  • 동기화 아키텍처 완성(이전 빌드 포함) — 파일(.md)이 원본, 오프라인 우선, 외부 편집(Obsidian/Files) 허용, 충돌 = 양쪽 보존(keep-both, 데이터 손실 0).

테스트 체크리스트

  • 첫 실행: 앱을 완전히 종료 후 다시 켰을 때 흰 화면이 오래 멈추지 않고 즉시 화면(셸→리스트)이 뜨는가.
  • 글 열기: 오래된/오프라인 글을 탭했을 때 무한 스피너 없이 바로 열리는가(비행기 모드로도 시도).
  • 편집→뒤로→재오픈(빌드 4 수정 핵심): 글 열기 → 편집 → 뒤로 → 같은 글 다시 탭 → 무한 스피너 없이 바로 열리는가(여러 번 반복).
  • 당겨서 새로고침(빌드 5 수정 핵심): 리스트를 아래로 당겨 새로고침을 반복해도(특히 앱 켠 직후) 느려지거나 멈추지 않는가.
  • 타이핑: 긴 글을 빠르게 입력할 때 키 입력이 끊기거나 뒤늦게 따라오는 랙이 없는가.
  • 외부 편집: 다른 기기/Files·Obsidian에서 같은 글을 고친 뒤 앱으로 돌아오면 변경이 반영되는가(앱이 백그라운드였다 복귀하는 경우 포함).
  • 충돌: 오프라인에서 편집·저장 → 그 사이 다른 기기에서 같은 글 편집 → 온라인 복귀 시 양쪽이 모두 보존되는가(데이터 손실 없음).
  • Settings/캡처: 설정 화면과 새 캡처 시트를 열 때 멈칫거림(행)이 없는가.
